# Функции и процессы

Существует два уровня описания функций и процессов: **короткий** (быстрый) и **подробный** (полный).  
Отличие только в синтаксисе — по факту создаётся один и тот же объект.

---

## 1. Объявление

### 1.1. Короткий стиль
Параметры перечисляются «как есть», типы и слоты подбираются автоматически.  
Поддерживаются **только одиночные** параметры.

```ts
// def и proc - это короткий стиль

def sum(a, b) -> 1:          // функция
    return num("a + b")

proc print(text):        // процесс
    player::message(text)
```

### 1.2. Подробный стиль
Каждому параметру явно указывается:
- тип (`one` / `multi` / `enum`)
- имя
- отображаемое имя (для GUI)
- слот и слот описания
- значение по умолчанию и тип

# Оформление функций и процессов: метаданные и параметры

После имени функции/процесса можно добавить **мета-блок** `<…>` — он задаёт:
- отображаемое название (в GUI редактора)
- иконку (предмет)
- краткое описание

Синтаксис:
```
def имя<"Отображаемое название", item("diamond"), "Описание">(параметры):
```
Аналогично для `proc`, `function`, `process`.

---

## 1. Мета-блок (одна строка)

| Позиция | Тип | Описание |
|---------|-----|----------|
| 1 | text | Название, которое увидит игрок в редакторе |
| 2 | item | Предмет-иконка (любая фабрика `item()`) |
| 3 | text | Краткое описание (тултип) |

**Пример:**
```ts
def give_kit<"Выдать набор", item("chest"), "Выдаёт стартовые вещи">(p, kit) -> 1:
```

---

## 2. Подробные параметры (полный формат)

Каждый параметр пишется в виде:
```
тип_параметра: имя<"отображаемое", слот(ы) [, флаги]>: тип_данных = значение_по_умолчанию
```

| Часть | Обязательна | Описание |
|-------|-------------|----------|
| `тип_параметра` | ✅ | `one` / `single` / `singular` <br> `multi` / `plural` <br> `enum` |
| `имя` | ✅ | техническое имя переменной |
| `"отображаемое"` | ✅ | текст для GUI |
| `слот(ы)` | ✅ | номер слота или массив слотов инвентаря-конфига |
| `флаги` | ❌ | зависят от типа (см. ниже) |
| `тип_данных` | ✅ | `number`, `variable`, `text`, `item`, … |
| `значение_по_умолчанию` | ❌ | значение, которое подставится, если пользователь ничего не выберет |

---

### 2.1. singular (одиночное значение)

Самая простая форма: один слот, одно значение.

```
singular: radius<"Радиус", 13>: number = 5
```

---

### 2.2. plural (список значений)

Принимает **массив**.  
Можно указать **игнорировать пустые** (`"false"`) — последний необязательный аргумент.

```
plural: items<"Предметы", [11,12,13], [20,21,22], "false">: variable = ["sword", "bow"]
```

| Позиция | Значение |
|---------|----------|
| `[11,12,13]` | слоты-входы |
| `[20,21,22]` | слоты-описания |
| `"false"` | не пропускать пустые (опц.) |

---

### 2.3. enum (выбор из списка)

Передаёт **индекс** выбранного варианта.  
Каждый вариант может иметь своё отображаемое имя и иконку.

```
enum: mode<"Режим", 7, 0>: [
    survival<"Выживание", item("iron_sword")>,
    creative<"Креатив", item("grass_block")>,
    spectator<"Наблюдатель", item("eye_of_ender")>
]
```

| Позиция | Значение |
|---------|----------|
| `7` | слот выпадающего списка |
| `0` | индекс значения по умолчанию (если невалидный — берётся 0) |

Формат одного значения:
```
имя_значения<"Отображаемое имя", item(...)>
```
Иконка и описание **опциональны**:
```
easy<"Лёгко">, normal, hard<"Сложно", item("netite_sword")>
```

---

## 3. Примеры «в одно окно»

```ts
function enchant_item<"Зачаровать", item("enchanted_book"), "Применяет выбранные чары">(
    singular: item<"Предмет", 10>: item,
    enum: level<"Уровень", 11>: [1, 2, 3] = 2,
    plural: enchants<"Чары", [12,13,14], [22,23,24]>: variable = []
) -> 1:
    // тело функции
```

---

## 2. Вызов

### 2.1. Функции
Возвращают результат(ы).  
Можно вызвать «по имени» или «через переменную» (без аргументов).

```ts
// обычный вызов
call function sum(3, 7) -> result
player::message("3 + 7 = $<result>")

// через переменную (аргументы не передаются)
call function (func_ptr)
```

### 2.2. Процессы
Дополнительно можно указать:
- режим копирования локальных переменных (`COPY`, `DONT_COPY`, `SHARE`)
- способ выбора цели (`CURRENT_SELECTION`, `CURRENT_TARGET`, `FOR_EACH_IN_SELECTION`, `NO_TARGET`)

```ts
call process heal(player, 5)                    // defaults
call process<"SHARE", "FOR_EACH_IN_SELECTION">  // только настройки
call process (heal_proc)                        // по указателю, без аргументов
```

---

## 3. Возвращаемые значения

Если функция возвращает значения, вы можете указать их количество после стрелки:

```ts
def swap(a, b) -> 2:
    return b, a
```

При вызове заберите столько же переменных:

```ts
call function swap(x, y) -> new_x, new_y
// new_x = y, new_y = x
```
