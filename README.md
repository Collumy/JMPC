
# JMPC - JustMc Python Compiler

## Установка
Для работы с JMPC нужно:

1. Открыть [страницу релизов GitHub](https://github.com/Collumy/JMPC/releases)
2. Выбрать понравившийся файл с компилятором
3. Открыть командную строку в папке с установленным файлом:\
   Win + R на **Windows**, затем вписать `cmd` в появивщееся окно
4. Использовать компилятор:

   ```shell
   # Для Windows
   $ py main.py example.jc
   ```

## Использование

_Перед тем как использовать `jmpc`, убедитесь что вы знаете [синтаксис](#синтаксис) для создания программ.\
Так же для использования jmpc, необходим установленный [python](https://www.python.org/downloads/) и модуль [requests](https://requests.readthedocs.io/en/latest/user/install/#install)_


Создайте файл с расширением `.jp` и таким содержанием:

```typescript
event player_join:
   player::message("Привет!")
```

Скомпилируйте данный файл:

```pwsh
$ py main.py [имя файла].jp
```

При его использовании, компилятор не будет создавать файлы компиляции на вашем компьютере, а сразу передавать их в облако. Если компиляция была успешной, у вас в терминале должна появиться ссылка, которую можно использовать на сервере напрямую.

## Синтаксис

### Значения

```ts
"Текст!"      // Текст
1             // Число
[1, 2]        // Список
{"a":1,"b":2} // Словарь

value("health") // Игровое значение без селектора
value("health", "current") // Игровое значение с селектором

item("stone") // Значения, созданные фабриками

```
#### Текст
```ts
l"текст" //текст legacy(цветной), по умолчанию любой текст вляется legacy
m"текст" //текст minimessage(стилизуемый)
p"текст" //текст plain(без всего)
j"текст" //текст json

```
#### Фабрики

Фабрики — функции, которые принимают аргументы и возвращают значение.
Ими создаются предметы, зелья, локации и другие значения, которые можно определить в коде:

```ts
a = location(1, 2, 3);
b = item("stone_sword", nbt={"minecraft:damage": 1})
```

### Переменные

Переменные можно использовать **без явного указания области**, либо с явным префиксом.

##### Неопределённые переменные

Если область не указана, она выбирается по контексту:

- в **событиях** — переменная считается **локальной**
- в **функциях/процессах** — переменная считается **строчной** (line)

```ts
varName = 5
`varName` = 10
```

> Обратные кавычки `` `name` `` позволяют использовать пробелы, спецсимволы и динамические имена.

##### Явно определённые переменные

**Строчные (line‑scope)**  

```ts
i`varName` = 5
line`varName` = 5
```

**Локальные (local‑scope)**  

```ts
l`varName` = 5
local`varName` = 5
```

**Игровые (game‑scope)**  

```ts
g`varName` = 5
game`varName` = 5
```

**Сохранённые (save‑scope)**  

```ts
s`varName` = 5
save`varName` = 5
```

## Плейсхолдеры

Плейсхолдеры позволяют выполнять вычисления и подстановки прямо внутри текста и имён переменных.  
Они записываются в виде:

```ts
$<выражение>
```

Например:

```ts
player::message("HP: $<math(hp + 5)>")

// Для чисел: num("выражение") - превращается в %math(выражение)
a = num("5 + 3 * 2")   // → %math(5+3*2)
```

Внутри `$<…>` работает **мини‑язык выражений**, полностью переработанный и расширенный по сравнению с обычными `%math()`.

### ⚙ Преобразования внутри плейсхолдера

Внутри `$<…>` компилятор автоматически исправляет и нормализует выражения.

#### Замены

| Ввод | Превращается в |
|------|----------------|
| `math(x)` | `%math(x)` |
| `value(id)` | `%val(id)` |
| `val(id)` | `%val(id)` |



### 📈 Улучшения математических функций

#### `min()` и `max()`  
Теперь поддерживают **неограниченное количество аргументов**.

```ts
min(1, 2, 3, 4)
// == 
min(1, min(2, min(3, 4)))
```

#### `clamp(x, min, max)`  
Ограничивает число в пределах:

```ts
clamp(2, 4, 8)
// ==
min(max(2, 4), 8)
```

---

### 🧩 Специальные плейсхолдеры переменных

Внутри `$<…>` можно использовать переменные всех областей.  
Они автоматически превращаются в корректные `%var`, `%index`, `%entry` и т.д.

| Запись в JP | Превращается в |
|-------------|----------------|
| g\`name\` | `%var(name)` |
| s\`list\`[index] | `%index_save(list, index)` |
| i\`dict\`{key} | `%entry_line(dict, key)` |
| len(smth) | `%length_local(smth)` |

Примеры:

```ts
player::message("Coins: $< g`coins` >")

player::message("Первый элемент: $< s`items`[0] >")

player::message("Размер списка: $< len(l`inventory`) >")
```

## События

[📓 Список событий](./documentation/starts/event.md)

Все действия, созданные вне определения события, будут добавлены к событию **запуска мира**, поэтому код:

```ts
player::message("Привет!");
```

**Никогда не сработает**, так как при запуске мира в нём нет игроков. Такой код нужно переделать на:

```ts
event player_join:
  player::message("Привет!")
```

Теперь при входе игрока ему будет писаться сообщение "Привет!".

Другие события определяются аналогичным способом, только айди события `player_join` будет изменяться на [📓 другое нужное](./documentation/starts/event.md).

## Действия

Вызов действий должен происходить от объекта или переменной.

_Анатомия вызова:_

```ts
  player::message("Привет мир!")
//^^^^^^
//переменная или объект

player::message("Привет мир!")
//    ^^
//    разделитель, определяющий тип вызова:
//    :: для объектов
//    . для переменных

player::message("Привет мир!")
//      ^^^^^^^
//      имя функции

player::message("Привет мир!")
//              ^^^^^^^^^^^^^
//              аргумент вызова
```

Аргументы вызова могут определяться:

- позиционно

  ```ts
  player::message("Привет мир!")
  ```

- с именем

  ```ts
  player::message(messages = "Привет мир!")
  ```

- комбинированно

  ```ts
  player::message("Привет мир!", merging = "SPACES")
  
  //Позиционные аргументы устанавливаются на следующее неустановленное значение:
  
  player::message("SPACES", messages = "Привет мир!")

  ```
Так же у действий есть селектор:
```ts
player::message<all>("Привет мир!")

//Выведет "Привет мир!" всем игрокам.
```

Действия, принимающие аргумент списка могут принимать одно его значение, и автоматически обворачивать его в список:

```ts
player::message("Привет мир!")
//              ^^^^^^^^^^^^^
//              Действие player::message принимает список,
//              но ему можно передать одно значение

player::message(["Привет", " ", "мир!"])
//              ^^^^^^^^^^^^^^^^^^^^^^^
//              Так тоже работает
```

Действия, принимающие **маркера** являются перечислениями, которым можно указать текст в виде значения:

```ts
player::message(["Привет", "мир"], merging = "SEPARATE_LINES")
```

##### Переменные

От переменных можно вызвать некоторые действия, которые принимают переменные _(либо значения, которые изменяют)_:

_Действие принимает переменную_

```ts
a = 1
a.increment(1)
```

_Действие принимает значение, которое изменяют_

```ts
a = ['Привет', 'мир']
a.set_list_value(1, 'сервер')
```

Существуют и действия, которые устанавливают переменным какое-либо значение:

```ts
a = variable::create_list([1, 2])
// Эквивалентно:
// a = [1, 2]

C = 3;
D = C.clamp(1, 2)
// Переменная D будет равна результату применения
// действия clamp к переменной C
```

[📓 Все действия](./documentation/actions/)

##### Объекты

Объекты аналогичны категориям действий: **действия игрока** (`player`), **действия сущности** (`entity`), **действия мира** (`world`), **действия с переменными** (`variable`)

_Вызов действия от объекта:_

```ts
player::message("Привет мир!")
variable::get_all_coordinates(location=location(1, 2, 3), x, y, z)
player::message("Привет мир спустя 10 секунд!")
```

## Кастомные действия

JP поддерживает **кастомные действия**, которые позволяют работать с типами напрямую:

- `text` / `t`
- `number` / `num` / `n`
- `list` / `l`
- `dict` / `map` / `d`

Каждый тип имеет собственный набор действий.

#### 📝 Кастомные действия: TEXT (`text` / `t`)

Используются для работы с текстами.

```ts
text: result = "Привет" + " " + "Джаст!"
// Установить текст в переменную

t: result = my_list.join(", ")
// Объединить список в текст
```


#### 🔢 Кастомные действия: NUMBER (`number` / `num` / `n`)

Используются для работы с числами.

```ts
num: a++
// Прибавление (a += 1)

n: num = floor(num)
// Округление в меньшую сторону
```


#### 📋 Кастомные действия: LIST (`list` / `l`)

Используются для работы со списками.

```ts
list: value = g`var`[10]
// Получение значения из списка

l: myList.append(1, 2, 3, 4, 5)
// Добавление значний в список
```

#### 🗂 Кастомные действия: DICT (`dict` / `map` / `d`)

Используются для работы со словарями.


```ts
map: i`length` = len(g`myMap`)
// Получение размера словаря

d: list.sort()
// Отсортировать словарь
```

## Функции и процессы

Два синтаксиса: короткий (`def`/`proc`) и подробный (`function`/`process`).  
Разница — только в способе описания параметров.

---

#### 1. Короткий стиль
Перечисляем параметры через запятую, слоты и типы подставляются автоматически.

```ts
def sum(a, b, c: variable):          // функция
    c = num("a + b")

proc print(text):                    // процесс
    player::message(text)
```

#### 2. Подробный стиль
Каждый параметр = тип + имя + GUI-данные + слоты + тип данных + значение по умолчанию.

```ts
function sum(
    singular: a<"1 число", 12, 21>: number = 0,
    single: b<"2 число", 14, 23>: number = 0,
    one: result<"Результат", 10, 19>: variable
):
    variable = num("a + b")
```

---

### Описание функций 
Работает **в любом стиле**:
```ts
def имя<"Название в GUI", item("diamond"), "Описание">(параметры):
```
| Позиция | Назначение |
|---------|------------|
| 1 | Название в GUI |
| 2 | Иконка (`item()`) |
| 3 | Описание |

---

#### Типы параметров (алиасы)

| Тип | Алиасы | Что передаёт | Обязательные поля |
|-----|--------|--------------|-------------------|
| **singular** | `single` / `one` | одно значение | `имя<"отображаемое", слот, слот_описания>` |
| **plural** | `multi` | массив значений | `имя<"отображаемое", [слоты], [слоты_описания]>` |
| **enum** | `marker` | индекс выбранного варианта | `имя<"отображаемое", слот>: [вариант, вариант]` |

---

### Вызов функций и процессов

#### Функции
```ts
call function sum(3, 7, l`result`)
call function (func_ptr)        // переменная-указатель, без аргументов
```

#### Процессы
Дополнительные опции в `<>`:
- режим локальных переменных: `COPY` / `DONT_COPY` / `SHARE`
- способ выбора цели: `CURRENT_SELECTION` / `CURRENT_TARGET` / `FOR_EACH_IN_SELECTION` / `NO_TARGET`

```ts
call process heal(player, 5)
call process<"SHARE", "FOR_EACH_IN_SELECTION"> heal(player, 5)
call process (heal_proc)
```

---

### Кастомный return
Укажите количество возвращаемых значений после стрелки:
```ts
def swap(a, b) -> 2:
    return b, a
```

При вызове заберите столько же переменных:
```ts
call function swap(x, y) -> new_x, new_y
```

## Импортирование

Не стоит сваливать весь код в один файл. Разделяйте свой код на несколько модулей, которые вы потом можете импортировать в один файл:

```ts
// variables.jp

a = 1;
b = 2;
```

```ts
// utils.jp

function say_hello:
  player::message("Привет!")
```

```ts
// index.jc

import variables.jp
from utils import say_hello

event player_join:
  player::message("Данный код состоит из 3 модулей!")

```
Логика импортирования заключается в том, что при импортировании файла, он компилируется сразу же.
При повторном импортировании того же файла, кол-во действий для загрузки на джаст не меняется, а данные для jmpc копируются.

## Определители действий

Если вам вдруг надоело писать player::message, или другие действия.
То вы можете их заменить с помощью `define`. Однако чтобы их использовать, вам нужно указать в начале строки `custom:`/`c:`

```ts
define msg = player::message

event player_join:
   c: msg("Привет")
```
